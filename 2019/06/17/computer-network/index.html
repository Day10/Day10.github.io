Chapter1 数制和编码

由于信息编码，IP地址格式转换，MAC地址格式转换需要了解数制的基本知识，所以有了本章内容。

  1. 数制是数据进制的简称，表示数据逢几进位的意思

    • 最新的的IPv6地址采用的是十六进制，IPv4用的是十进制表示。
    • N进制的最大数码是基数减一，最小数码是0.比如十六进制，最大数码是16-1=15（为F），最小为0.

  2. 数据编码用于以不同形式表示数据，主要包括原码，反码，补码三种

    • 源码，反码，补码和移码都是计算机二进制数（机器数）的不同表现形式，之所以有这么多码，是因为计算机只有加法运算器。减法要转换成负数的加法。
    • 字长指计算机一次可以处理二进制数的码位长度，是计算机进行数据存储和数据处理的运算单位。字长越长表示计算机处理能力越强
    • 原码就是数对应二进制本身多代表的形式。 +3 = 00000011
    • 补码，正数的补码与原码相同，负数的补码是，先将除符号位以外的其他位取反，然后在末位加一得到的。补码是为了方便实现减法。

Chapter2 计算机网络概述

定义：指将不同地理位置，具有独立功能的多台计算机及网络设备通过通信线路（包括传输戒指和网络设备）连接起来，在网络操作系统、网络管理软件及网络通信协议的共同管理和协调下实现资源共享和信息传递的计算机系统。（额……）

计算机网络发展史

1. 第一代：以计算机主机为中心，一台或多台终端围绕计算机主机分布在各处。计算机主机进行成批处理，终端不具备数据储存和处理功能。缺点是计算机主机负荷重，并且如果主机瘫痪会使计算机网络瘫痪。
2. 第二代：分组交换式的计算机网络。分成资源子网和通信子网。资源子网是又各个主机组成，通信子网由IMP（接口报文处理机）组成。主机间通信是通过IMP连接起来的网络实现。缺点是没有统一的网络体系架构和协议标准，各公司间网络不能相连，不利于厂商和用户。
3. 第三代：标准化的计算机网络。发布OSI/RM模型，定义网络互连的七层结构和功能。
4. 第四代：国际化的计算机网络。以Internet代表的因特网的发展。
5. 第五代：猜想主要特征为“三网”融合，物联网，虚拟化，云计算，HTML5。

计算机网络的基本组成

  1. 网络硬件系统
    • 计算机设备：由网络用户控制和使用的计算机（PC，iPAD等）
    • 网络连接和通信设备： 除计算机之外的设备（网卡，网桥等）
    • 传输介质：简单说就是网线，是网络通信的“路”
  2. 网络软件系统：安装在终端计算机中，用于计算机网络通信和应用的计算机程序。
    • 网络应用平台（windows系统，UNIX系统等）
    • 网络通信协议（TCP/IP）
    • 具体网络应用的工具软件（QQ，FireFox）

计算机网络拓扑结构

拓扑（Topology）学是一种研究与大小、距离无关的几何图形特征的方法。
“网络拓扑结构”是由网络节点设备和通信介质通过物理连接所构成的逻辑结构图。

  1. 节点
    • 一个节点是一个网络端口
    • 分为‘转节点’和‘访问节点’
    • 转节点作用是支持网络连接，它通过通信线路（交换机、路由器）转接和传递信息
    • 访问节点是信息交换的源点和目标点（通常是网卡接口）
  2. 结点
    • 一个’结点‘是指一台网络设备，连接多个节点
    • 分为链路结点和路由结点，分别对应网络中的交换机和路由器
  3. 链路
    • 两个节点之间的线路
    • 分为物理链路和逻辑链路
    • 物理链路指实际存在的通信线路，由设备网络端口和传输介质连接实现
    • 逻辑链路指逻辑上起作用的网络通路，由数据链路层标准和协议实现
  4. 通路
    • 从发出信息的节点到接受信息的节点之间的一串节点和链路的组合
    • 可能包含多个链路

  5. 拓扑结构（简单）

    • 有线拓扑结构

      1. 星形拓扑结构（集中式拓扑结构）
      2. 集线器和交换机连接的各个节点呈星形（放射状）分布
      3. 有中央结点与其他与中央结点相连的其他节点组成
      4. 主要运用于有线以太局域网中
      5. 环形拓扑结构（现在基本不用了）
      6. 总线性拓扑结构
      7. 树形拓扑结构
      8. 网状拓扑结构
      9. 混合型拓扑结构

    • 无线局域网拓扑结构

      1. Ad-Hoc
      2. Infrastructure

Chapter 3 计算机网络体系结构

典型计算机网络体系结构

  1. 是一个分层次的模块式结构
  2. 在第三代计算机网络中，ISO推出了第一个标准化的计算机网络体系结构–OSI/RM（Open System Interconnection Reference Model）开放系统互连参考模型
  3. OSI/RM
    1. 分为七层
    2. 由低到高：物理层，数据链路层，传输层，网络层，会话层，表达层，应用层
    3. 广域网其实是多个远程局域网的连接，所以OSI/RM包括了广域网和局域网通信的功能层次
    4. 低四层，主要定义了如何进行端对端的数据传输。
    5. 高三层，定义了终端系统的应用程序如何彼此通信。
    6. 低三层，属于‘通信子网’，面向网络；主要打通了一条用于数据传输的网络通道。
    7. 高四层，属于‘资源子网’，面向用户；主要为具体的网络应用提供应用平台和端对端的数据传输通道。
    8. 并不是每一次通信都需要OSI的全部七层；但网络应用通信都要经过七层
    9. 网络体系中的每一层不是针对具体的设备和软件而言的，而是针对网络功能
    10. 遵循的规则
      1. 同层的各个网络节点有相同的层次结构和功能
      2. 同一节点内相邻层之间通过接口通信
      3. 每一层使用下一层提供的服务，并向上层提供服务
      4. 不同节点的同等层按照协议实现对等层之间的通信
      5. 见6，7点
  4. TCP/IP协议体系结构
    1. 针对TCP/IP协议簇的广域网开发的
    2. 分为四层
    3. 由低到高：网络访问层，网络互连层，传输层，应用层
    4. 和OSI/RM的对应
      1. 应用层 = 应用层 + 表示层 + 会话层
      2. 传输层 = 传输层
      3. 网络互连层 = 网络层
      4. 网络访问层 = 数据链路层 + 物理层
    5. TCP/IP五层模型（最佳）
      1. 应用层 = 应用层 + 表示层 + 会话层
      2. 传输层 = 传输层
      3. 网络互连层 = 网络层
      4. 数据链路层 = 数据链路层
      5. 物理层 = 物理层
  5. 局域网体系结构
    1. 有线以太局域网体系结构
    2. 只有两层
    3. 物理层和数据链路层
    4. 数据链路层分为LLC子层和MAC子层
